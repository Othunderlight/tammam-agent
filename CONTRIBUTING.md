# Contributing to FounderStack AI

Hello there! 👋 We're so glad you're interested in helping out. This project is built for founders and builders, and we want to make it as easy as possible for anyone to contribute, even if it's your first time in open source!

## 🌟 Our Philosophy: Keep it Simple

We believe in progress over perfection. You don't need to be a senior engineer to contribute. If you find a typo, a bug, or have a cool idea, we want to hear from you!

## 🛠️ How to Get Started

### 1. Be Kind
We follow the [Code of Conduct](CODE_OF_CONDUCT.md). Treat everyone with respect. No question is too "basic."

### 2. Talk First (For Big Changes)
If you want to add a major feature, please **open an Issue** first. This helps us discuss the approach together so you don't spend hours on something that might not fit the project's direction.

### 3. Fork and Branch
1.  **Fork** the repository to your own account.
2.  Create a new **branch** for your change (e.g., `git checkout -b fix-typo-in-readme`).
3.  Make your changes and commit them with a clear message.

### 4. Small is Beautiful
We prefer 5 small Pull Requests (PRs) over 1 giant, confusing one. Small PRs are easier to review and get merged faster!

## 🚀 Setting Up Your Environment

1.  Clone your fork: `git clone https://github.com/YOUR_USERNAME/fastai.git`
2.  Copy the example environment file: `cp .env.example .env`
3.  Fill in your API keys in the new `.env` file.
4.  Install dependencies: `pip install -r reqs.txt` (or use `uv run` if you prefer).

## ✅ Checklist for PRs

- [ ] Does it work? (Test it locally!)
- [ ] Is it simple? (Avoid over-complicating things.)
- [ ] Did you update the docs if needed?

---

**Never worked in Open Source before?** Don't worry! Ping us in your PR or an Issue, and we'll help guide you through the process. We're all here to learn together! 🚀
